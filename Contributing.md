# Contributing to CliniQ  Blueprint

Thank you for your interest in contributing to the **CliniQ  Blueprint** provided by Cloud2 Labs.

# Guidelines

- Fork the repo and create a feature branch for your changes

- Appropriate contributions include documentation improvements, bug fixes, reference architecture enhancements, and educational examples

- Major features or architectural changes require prior discussion with maintainers via GitHub Issues

- Follow existing standards for coding and documentation style

- Never include sensitive, proprietary, or regulated data (PHI, PII, credentials)

- Submit a pull request with a clear description of your changes and reference any related issues

- Test thoroughly before submitting to ensure no breaking changes

For questions or major proposals, please open an issue first. We appreciate your contributions!
